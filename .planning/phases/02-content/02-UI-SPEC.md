## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline gaps |
| sm | 8px | Between inline elements (CTA links row, skills tags) |
| md | 16px | Default element spacing (paragraph margin, list gaps) |
| lg | 24px | Component internal padding, nav item spacing |
| xl | 32px | Between sections within a page region |
| 2xl | 48px | CV entry vertical gaps, major intra-section breaks |
| 3xl | 64px | Reserved for padding around section headings |

Section-level vertical padding: `--space-section: clamp(4rem, 3rem + 5vw, 10rem)` — add to `src/styles/tokens.css`.

Card radius: `--radius-card: 0.75rem` — add to `src/styles/tokens.css`.

Exceptions:
- Touch target minimum: 44px height for all interactive links (CTA, nav links) per WCAG 2.5.5.
- Hero section: full viewport height via `min-h-screen` (not a spacing token, a layout constraint).

Source: 02-CONTEXT.md D-12, global coding-style.md spacing guidance, WCAG 2.5.5.

## Typography

| Role | Token | Size Range | Family | Weight | Line Height |
|------|-------|------------|--------|--------|-------------|
| UI / label | `--text-ui` | clamp(0.875rem → 1rem) | Sora | 400 | 1.4 |
| Body | `--text-body` | clamp(1rem → 1.125rem) | Sora | 400 | 1.6 |
| Heading (section title) | `--text-heading` | clamp(1.75rem → 2.5rem) | Instrument Serif | 400 | 1.2 |
| Display (hero name) | `--text-hero` | clamp(3rem → 7rem) | Instrument Serif | 400 | 1.0 |

Weights used: **400** (regular, used for everything) and **600** (semibold, used for nav active state, work entry role titles, and CTA hover emphasis only).

Note: Instrument Serif only ships weight 400 — no bold variant available or needed. All display type is set in regular weight by design intent.

Source: `src/styles/tokens.css` (all four tokens already defined), `src/app/layout.tsx` (Sora loads 400/500/600/700), `src/app/globals.css` (body line-height 1.6), 02-CONTEXT.md D-03.

## Color

All tokens are defined in `src/styles/tokens.css` using Tailwind v4 `@theme`. Phase 2 uses these tokens only — no new color tokens.

| Role | Token | Approximate Value | Usage |
|------|-------|-------------------|-------|
| Dominant (60%) | `--color-surface` | `oklch(97% 0.01 75)` ≈ warm off-white | Page background, header background (post-scroll), section backgrounds |
| Secondary (30%) | `--color-muted` | `oklch(55% 0.03 75)` ≈ warm gray | Body text de-emphasis, dates in CV timeline, company name in work entries, nav link default state |
| Accent (10%) | `--color-accent` | `oklch(62% 0.19 55)` ≈ warm amber/terracotta | Reserved for: hero name underline or decorative mark; active/hover nav link; CTA link hover underline; no other uses |
| Text (primary) | `--color-text` | `oklch(18% 0.01 75)` ≈ warm near-black | All body copy, headings, section labels, nav links |
| Destructive | n/a | n/a | No destructive actions in Phase 2 |

Accent reserved for (explicit list):
1. Hero name — a single short decorative underline or accent mark below the display name
2. Active nav link — the currently-in-view section link in the sticky header
3. CTA link hover state — underline color on hover for the four hero CTA links
4. Section label left border or marker in the CV two-column layout

Source: `src/styles/tokens.css`, 02-CONTEXT.md (warm parchment palette), design-quality.md (color used semantically), CLAUDE.md (light theme only).

## Layout Specifications

### Header
- Position: `position: sticky; top: 0; z-index: 50`
- Initial state: `background-color: transparent`
- Post-scroll state: `background-color: var(--color-surface)` with subtle `box-shadow`
- Scroll threshold: `window.scrollY > 100`
- Layout: name (`Axel W`) flush left, nav links flush right
- Mobile (< 640px): name only — nav links hidden
- Desktop (≥ 640px): full name + all four nav links (About, CV, Projects, Contact)
- Nav links: `font-family: var(--font-body)`, size `--text-ui`, weight 400 default / 600 active
- Header height: 64px

Source: 02-CONTEXT.md D-09, D-10, D-11, specifics section.

### Hero Section
- Height: `min-h-screen` (full initial viewport)
- Alignment: left-aligned — no centering
- Stack order (top to bottom): display name → title/tagline → bio (1–2 sentences) → CTA links row
- Display name: `--text-hero`, Instrument Serif weight 400, `--color-text`
- Title: `--text-body`, Sora weight 600, `--color-muted`
- Bio: `--text-body`, Sora weight 400, `--color-text`, max-width 60ch
- CTA links: `--text-ui`, Sora weight 400, four items in a row with 16px gaps — GitHub, LinkedIn, Email, CV Download
- Max content width: 800px, horizontally padded on mobile
- No photo, no decorative blob, no gradient background

Source: 02-CONTEXT.md D-01, D-02, D-03, D-04, REQUIREMENTS.md HERO-01 through HERO-03.

### CV Section
- Layout: two-column grid — left column 20% (section label), right column 80% (entries)
- Mobile (< 640px): single-column — section label stacks above entries
- Section labels: `--text-ui`, Sora weight 600, `--color-muted`, uppercase letter-spacing 0.08em
- Entry role/degree title: `--text-body`, Sora weight 600, `--color-text`
- Entry company/institution + dates: `--text-ui`, Sora weight 400, `--color-muted`
- Entry description: `--text-body`, Sora weight 400, `--color-text`, line-height 1.6
- Skills: flat tag list — comma-separated spans or small pill-shaped items, `--text-ui`, no progress bars
- Vertical gap between entries: 48px (2xl)
- Section padding: `var(--space-section)` top and bottom

Source: 02-CONTEXT.md D-06, D-07, REQUIREMENTS.md CV-01 through CV-04.

### Contact Section
- Four links: Email, LinkedIn, GitHub, CV Download
- Same visual treatment as hero CTA links for consistency
- Section padding: `var(--space-section)` top and bottom

Source: REQUIREMENTS.md CV-04, INFRA-04.

### Scroll-Reveal (FadeUp)
- Initial state: `opacity: 0; transform: translateY(16px)`
- Revealed state: `opacity: 1; transform: translateY(0)`
- Duration: 400ms
- Easing: `ease-out` (approximates `cubic-bezier(0.16, 1, 0.3, 1)`)
- Trigger: IntersectionObserver threshold 0.15
- Reduced motion: `opacity: 1; transform: none` immediately — no animation, instant appear

Source: 02-CONTEXT.md D-13, D-14, REQUIREMENTS.md INFRA-05, performance.md (compositor-only properties).

## Accessibility Contract

- All interactive elements have visible focus indicators (2px outline)
- Color contrast: `--color-text` on `--color-surface` = ~11:1 (passes WCAG AA and AAA)
- `--color-muted` on `--color-surface` = ~5.5:1 (passes WCAG AA)
- `--color-accent` on `--color-surface` = ~4.6:1 (passes WCAG AA)
- Semantic HTML: `<header>`, `<main>`, `<section aria-labelledby>`, `<nav aria-label>`, `<footer>`
- Scroll-reveal: respects `prefers-reduced-motion` — no animation when set
- Touch targets: minimum 44px height on all links

Source: REQUIREMENTS.md INFRA-05, WCAG 2.4.3, 2.5.5, coding-style.md (semantic HTML first).
