## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Committed JSON file → build pipeline | The fallback JSON is committed to git. A malicious commit could inject arbitrary repo names/descriptions/URLs into the rendered page. |
| Build host → GitHub REST API | Same as Plan 01; unchanged. |
| Build log → CI surface | A single intentional console.warn appears in build logs on the fallback path. Must not contain secrets. |
| Test fixture (placeholder GITHUB_TOKEN) → CI logs | An obviously-invalid placeholder token is injected to force a 401. Token format chosen so collision with any real token is impossible. |
