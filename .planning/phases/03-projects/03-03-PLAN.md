    --- src/lib/date.ts ---

    Create the file as a pure-utility module with a single named export. No `'use client'`, no React imports, no runtime side effects.

    Export signature: `export function formatRelativeDate(iso: string, now: Date = new Date()): string`

    Behavior contract:
    1. Parse `iso` via `new Date(iso)`. If parsing yields an Invalid Date (Number.isNaN(parsed.getTime())), return the empty string `''` and do nothing else (no throw — the caller renders nothing if the meta line ends up empty).
    2. Compute the millisecond delta `now.getTime() - parsed.getTime()`. If delta < 0 (the date is in the future, which can happen for newly pushed repos with clock skew), treat it as 0 → 'today'.
    3. If delta < 24 * 60 * 60 * 1000 (24 hours), return the literal string `'today'` (UI-SPEC override).
    4. Otherwise, compute the largest unit ≥ 1 from { year, month, day } using these conversion factors:
       - 1 year ≈ 365.25 days (use a const named DAYS_PER_YEAR with the value 365.25 — magic-number rule from coding-style.md)
       - 1 month ≈ 30.44 days (const DAYS_PER_MONTH = 30.44)
       Compute days = delta / (24*60*60*1000), then years = days / DAYS_PER_YEAR, months = days / DAYS_PER_MONTH.
       If Math.floor(years) ≥ 1: unit is 'year', value is -Math.floor(years) (negative because the event is in the past).
       Else if Math.floor(months) ≥ 1: unit is 'month', value is -Math.floor(months).
       Else: unit is 'day', value is -Math.floor(days).
    5. Construct `new Intl.RelativeTimeFormat('en', { numeric: 'auto' })` once per call (Intl.RelativeTimeFormat is cheap; no module-level cache needed for a portfolio site of this size — KISS rule).
    6. Return `formatter.format(value, unit)` — output examples: `'yesterday'`, `'3 days ago'`, `'2 months ago'`, `'last year'`, `'5 years ago'`.

    The defaulted `now` parameter exists so future tests can pin the clock without mocking globals. Plan 03 does not include unit tests for this utility (testing is Playwright per the project's testing.md), but the parameter is cheap to add and aligns with typescript coding-style.md "Public APIs" rule about explicit parameter and return types. The relative-date output format is exercised end-to-end by the Playwright UAT in Task 3 (UAT-1 includes a meta-line regex check), so a separate Node runtime smoke test is unnecessary.

    No `any`. No `console`. No imports outside built-ins.
