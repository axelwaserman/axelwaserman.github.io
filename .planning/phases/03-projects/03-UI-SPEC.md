# Phase 3 — UI Design Contract

> Visual and interaction contract for the Projects section. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Pre-populated source legend:** `[CTX]` = 03-CONTEXT.md, `[T1]` = Phase 1 tokens (`src/styles/tokens.css`), `[T2]` = Phase 2 component patterns (`src/components/cv/*`, `src/components/hero/*`, `src/components/contact/*`), `[CL]` = `CLAUDE.md`, `[default]` = sensible default applied where upstream was silent.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (token-driven) `[CL]` |
| Preset | not applicable — `CLAUDE.md` rules out shadcn ("fights the custom design direction; shortcircuits the learning goal") `[CL]` |
| Component library | none — handwritten components composed with Tailwind v4 utilities + CSS custom property tokens `[T1]` |

**Design direction (locked from prior phases):** Editorial, warm parchment, light theme only. Card visual must satisfy the anti-template policy (`web/design-quality.md`) — intentional hover/focus states, hierarchy via type scale and weight contrast (not via color spam), and a deliberate card surface that diverges from default Tailwind/shadcn card looks. `[CTX D-01]`

## Spacing Scale

Project uses a fluid, token-driven system rather than a strict 8-point ladder. The locked scale (Phase 1 + 2) is:

| Token | Value | Usage |
|-------|-------|-------|
| Tailwind 1 | 4px | Tight inline gaps inside meta line |
| Tailwind 2 | 8px | Skill-pill padding, chip-style tag gaps `[T2]` |
| Tailwind 3 | 12px | Card internal vertical rhythm (label → title) |
| Tailwind 4 | 16px | Card internal padding default, link-row gap `[T2]` |
| Tailwind 6 | 24px | Card padding (`p-6`), grid `gap-6`, page horizontal padding `px-6` `[T2]` |
| Tailwind 8 | 32px | Section heading-to-content separation |
| Tailwind 12 | 48px | Wide-gutter grid breathing room (desktop) |
| `--space-section` | clamp(4rem, 3rem + 5vw, 10rem) | Section vertical padding (`py-[var(--space-section)]`) `[T1]` |
| `--radius-card` | 0.75rem (12px) | All card corners (already in use by `SkillsList` chips) `[T1]` |
| `scroll-mt-16` | 64px | Anchor-link offset under the 64px sticky header `[T2]` |

**Section container pattern (locked from `[T2]`):** `py-[var(--space-section)] px-6 max-w-4xl scroll-mt-16` — every top-level section, including this one, uses identical outer chrome.

**Card grid:**
- Mobile (<640px): single column, `gap-6` (24px) — `[CTX D-01]`
- Tablet/desktop (≥640px): two columns, `gap-6` (24px) `[default — keeps to existing 4xl max-width]`
- Desktop (≥1024px): two columns retained (do **not** push to three; `max-w-4xl` would crush a 3-col layout into <300px tracks) `[default]`

**Card internal:** `p-6` (24px all sides). Title-to-meta gap: 4px. Meta-to-description gap: 12px. Description-to-link-row gap: 24px. `[default — derived from CV/WorkEntry rhythm]`

**Exceptions:** none. All spacing values are multiples of 4 and use existing tokens or Tailwind step classes.

## Typography

Sizes pulled directly from `[T1]` `tokens.css` clamp() scale. Phase 3 uses **3 of the 4** roles defined globally (no `--text-hero` here — that is Hero-only). Weights: regular (400) + semibold (600), matching `[T2]` precedent.

| Role | Token | Computed range | Weight | Line height | Usage in Phase 3 |
|------|-------|----------------|--------|-------------|------------------|
| Section heading | `--text-heading` | clamp(1.75rem, 1.4rem + 1.75vw, 2.5rem) → 28-40px | 400 (Instrument Serif) | 1.1 | "Projects" h2 |
| Card title (repo name) | `--text-body` | clamp(1rem, 0.92rem + 0.4vw, 1.125rem) → 16-18px | 600 (Sora) | 1.3 | Repo name h3 inside each card |
| Body / description | `--text-body` | 16-18px | 400 (Sora) | 1.6 (inherited from `body`) | Repo description |
| Meta / UI | `--text-ui` | clamp(0.875rem, 0.82rem + 0.28vw, 1rem) → 14-16px | 400 (Sora, semibold for label) | 1.5 | Language, last-pushed date, link labels, section eyebrow |

**Heading style:** Section heading uses `font-[var(--font-heading)]` (Instrument Serif) — same treatment as Contact's "Get in touch" `[T2]`.

**Link/CTA typography:** Text-only links, no buttons. `text-[length:var(--text-ui)]`, `underline-offset-2`, `decoration-[var(--color-muted)]`, hover swaps to `decoration-[var(--color-accent)]` and `text-[var(--color-accent)]`. **Identical to Hero/Contact CTAs** `[T2]` — keeps the link vocabulary consistent across the page.

## Color

Single-theme, light only. Tokens locked in `[T1]`. The 60/30/10 split below maps to the warm parchment palette already shipped.

| Role | Token | OKLCH value | Usage in Phase 3 |
|------|-------|-------------|------------------|
| Dominant (60%) | `--color-surface` | `oklch(97% 0.01 75)` | Page background; section background (inherits from body); also card surface (cards are flat-on-surface, distinguished by 1px hairline border, not by a fill swap — see card surface treatment below) |
| Text / Foreground (30%) | `--color-text` | `oklch(18% 0.01 75)` | Section heading, card title, description body |
| Muted / Secondary (within 30%) | `--color-muted` | `oklch(55% 0.03 75)` | Eyebrow label ("Projects"), meta line (language · pushed-date), default link decoration, hairline borders at `/20` opacity |
| Accent (10%) | `--color-accent` | `oklch(62% 0.19 55)` | **Reserved for:** (1) link hover/focus text + decoration color, (2) focus ring outline (`focus:outline-[var(--color-accent)]`), (3) optional subtle card-hover edge highlight (border color swap on `:hover` of the article element). **Not used for:** card backgrounds, card titles in default state, description body, meta text, or any decorative fill. |
| Destructive | not declared | n/a | No destructive actions in Phase 3. Read-only data display. |

**Accent reserved for (explicit list — must match exactly):**
1. Hover state on repo link text + underline
2. Hover state on live-demo link text + underline
3. Focus ring on any focusable link (`focus:outline-2 focus:outline-[var(--color-accent)] focus:outline-offset-2`)
4. Optional: card border hover highlight (border swaps from `border-[var(--color-muted)]/20` → `border-[var(--color-accent)]/40`) — compositor-friendly, supports anti-template hover requirement
5. Optional: card-title hover color swap (when the entire card is hoverable, the repo name turns accent — same vocabulary as a link)

**Card surface treatment (key anti-template move):**
- Default: `bg-[var(--color-surface)]` (same as page) + `border border-[var(--color-muted)]/20` + `rounded-[var(--radius-card)]`
- This produces a tonally-flat card that reads as a **boundary**, not a chip. The hierarchy comes from the title weight (600) + warm hairline border, not from a contrasting fill — which is the editorial direction the anti-template policy demands.
- Optional secondary surface variant if depth is needed at the planner's discretion: a subtler warm tint card surface using `oklch(95% 0.012 75)` (one step darker than `--color-surface`). This is reserved as fallback only if the hairline-on-flat treatment looks insufficiently differentiated in implementation review.

**Contrast verification (must hold):**
- `--color-text` on `--color-surface`: ≥ 12:1 (passes WCAG AAA body) — verified via prior phases.
- `--color-muted` on `--color-surface`: ≥ 4.5:1 (passes WCAG AA body) — verified via prior phases.
- `--color-accent` on `--color-surface`: must pass ≥ 4.5:1 — checker should re-verify if any accent-on-surface text is used (currently only on hover, where momentary contrast can dip slightly but not below AA).

## Copywriting Contract

Voice: terse, factual, no marketing copy. Plain-noun link labels match Hero/Contact precedent (`"GitHub"`, `"LinkedIn"`, `"Email"`, `"Download CV"`). `[T2]`

| Element | Copy | Rationale |
|---------|------|-----------|
| Section eyebrow label | `Projects` (UPPERCASE, `tracking-[0.08em]`, `font-semibold`, `--color-muted`) | Mirrors CV "Work" / "Education" / "Skills" eyebrow pattern `[T2]`. Keeps the page's typographic system consistent. |
| Section heading (visible h2 — see structural note) | `Selected work on GitHub` | Editorial, specific. Avoids the generic word "Projects" twice. The eyebrow already carries the section name; the heading carries voice. `[default]` |
| Card title (repo name) | `{repo.name}` (verbatim from API, no transformation) | Names like `gsd-sdk` and `axelw.github.io` are intentional. Do not title-case. `[default]` |
| Card meta line format | `{language} · {relative_date}` — e.g. `TypeScript · 3 months ago`. If `language` is null, render only `{relative_date}`. Use `·` (U+00B7 middle dot) as separator with single spaces. | `[default — derived from CTX D-07]` |
| Description fallback (when GitHub `description` is null/empty) | Render no description text at all (omit the `<p>` element entirely; do not show "No description" or em-dash placeholder). | `[default]` Empty placeholders read as broken; absence reads as intentional. |
| Repo link label | `Repo` (with trailing arrow glyph `→` rendered as a separate inline element for accessibility — visual only, hidden from screen readers via `aria-hidden="true"`) | Plain noun, matches Hero "GitHub" / "LinkedIn" `[T2]` voice. Arrow signals external nav. |
| Live demo link label (only when `homepage` set) | `Live demo` (with trailing `→`) | `[default]` — clear noun phrase, matches voice. |
| Relative date format | `Intl.RelativeTimeFormat('en', { numeric: 'auto' })` outputs (e.g., `yesterday`, `3 months ago`, `2 years ago`). For dates < 24h, output `today`. | `[CTX Claude's discretion]` — chosen because it is built-in, zero deps, and locale-correct. |
| Empty state heading (zero non-archived repos returned, after fallback) | `No public projects yet.` | `[default]` — terse, no apology, no marketing. |
| Empty state body | `Check back soon — or look at the source for [this site itself](https://github.com/axelw/axelw.github.io).` | `[default]` — converts the dead end into a useful link (this site's own repo is always a valid project). The link uses the same hover vocabulary as all other links. |
| Error state | not exposed to user | `[CTX D-06]` On API failure during `next build`, the build falls back to `src/data/projects.json` and the page renders normally. A console warning is logged at build time. The user never sees an error UI. |
| Destructive confirmation | n/a | No destructive actions. Read-only section. |

**Structural note on heading visibility:** Phase 2 sections currently follow two patterns — Hero/Contact have visible h-level headings; CV uses an `sr-only` h2 because its eyebrow labels carry the role visually. Phase 3 should use a **visible h2** matching the Contact pattern (single-section content with one cohesive theme), not the CV `sr-only` pattern. This avoids the eyebrow-only-no-heading visual that would look odd against a card grid and gives the section more editorial presence.

## Interaction & Motion Contract

| State | Behavior |
|-------|----------|
| Default | Card displays with hairline border, name in 600 weight, meta in muted color, links underlined in muted decoration. |
| Hover (entire card) | Border swaps from `--color-muted/20` → `--color-accent/40` over 200ms ease-out. Repo-name h3 swaps to `--color-accent`. The two text links inside the card swap independently (their own hover states). The card itself does **not** lift, scale, or shadow — keep the editorial-flat aesthetic. |
| Focus (focusable link only) | `focus-visible:outline-2 focus-visible:outline-[var(--color-accent)] focus-visible:outline-offset-2`. Same as Hero/Contact. |
| Active (mouse-down on link) | Browser default — no custom active state needed. |
| Reduced motion | All transitions still apply (they are color/border, not transform/opacity, but ≤ 200ms color transitions are within prefers-reduced-motion's "fine" envelope). The `<FadeUp>` wrapper handles the section-entry reveal correctly per its existing implementation. |
| Keyboard navigation | Each link is independently focusable and announceable. Card itself is **not** wrapped in a single big `<a>` — that would steal focus from inner links. Use two visible inline links per card; let users tab between them. |

**Compositor-only animations:** Border-color and color transitions are paint-only, not layout-affecting — within the project's performance budget per `web/performance.md`.

## Accessibility Contract

- Section uses `<section id="projects" aria-labelledby="projects-heading">`.
- Visible h2 carries `id="projects-heading"`.
- Each card is an `<article>`. Card title is `<h3>`.
- Card meta line uses semantic `<dl>` only if it carries 2+ key-value pairs; otherwise plain text suffices. **Default: plain text** — language and date are tertiary metadata, not a definitions list.
- Card grid wrapper is a `<ul>` with each card as `<li>` containing the `<article>` — this gives screen-reader users a clear "list of N items" announcement. (CV's `SkillsList` already uses `<ul>/<li>` for the same reason.) `[T2]`
- Repo link: `aria-label` = `"View {repo.name} repository on GitHub"`. The visual `→` glyph is `aria-hidden="true"`.
- Live demo link: `aria-label` = `"View {repo.name} live demo"`.
- All external links: `target="_blank" rel="noopener noreferrer"` (matches Hero/Contact pattern) `[T2]`.
- Color contrast holds at all states (see Color section verification).
