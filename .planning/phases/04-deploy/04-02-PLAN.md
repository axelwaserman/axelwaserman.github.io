GitHub Actions cron syntax (POSIX-cron, UTC-only):
- 5 fields: minute hour day month day-of-week
- `'0 6 * * *'` = every day at 06:00 UTC
- The cron string MUST be quoted in YAML (the `*` characters trigger YAML alias errors otherwise)
- GitHub Actions schedules run on UTC — no timezone option exists

workflow_dispatch syntax:
- Bare `workflow_dispatch:` (no inputs) is valid and adds a "Run workflow" button to the Actions UI
- We do not need `inputs:` for this plan — manual run with no parameters is the only need

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cron scheduler -> deploy.yml | GitHub Actions cron triggers run with the same `secrets.GITHUB_TOKEN` scope as a push-triggered run. No additional trust surface. |
| Repo collaborator with write access -> workflow_dispatch | A workflow_dispatch trigger is gated by the same write-access check as `push: main`. Anyone who can push to main can already trigger a deploy implicitly; adding the manual button does not expand the trust circle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Denial of Service | Cron + push triggers racing each other | mitigate | The Plan 01 `concurrency: group: pages, cancel-in-progress: false` block (unchanged in this plan) serializes overlapping runs. A cron run kicked off at 06:00 UTC while a push deploy is mid-flight will queue and run after, not race. |
| T-04-07 | Tampering | Cron schedule drift / accidental misfires | accept | A daily cron at 06:00 UTC has a worst case of one stale-by-24h projects list. The blast radius is "site shows yesterday's pushed_at sort order" — fully acceptable. No mitigation needed. |
| T-04-08 | Elevation of Privilege | workflow_dispatch as a backdoor for non-main code | accept | workflow_dispatch defaults to running from the default branch (main). To run from a non-default branch, GitHub Actions requires explicit `inputs:` — we did not declare any. The button can only run main. |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages added. The Plan 01 supply-chain disposition carries forward. |

</threat_model>
