Pure utility module shape (analog: src/lib/date.ts — see PATTERNS.md §"src/lib/mandala.ts (NEW — utility)"):

  export interface ChordLine { x1: number; y1: number; x2: number; y2: number }
  export interface MandalaPair { n: number; k: number }
  export const CURATED_PAIRS: readonly MandalaPair[]
  export function generateLines(n: number, k: number): ChordLine[]
  export function pickRandomPair(exclude?: MandalaPair, rng?: () => number): MandalaPair

Algorithm (UI-SPEC §"Algorithm (D-03)"):
  1. Place n points on a unit circle: point i at angle θ_i = (2π * i) / n − π/2 (start at 12 o'clock, sweep clockwise).
  2. For each i in [0, n): j = (i * k) mod n.
  3. Emit ChordLine from point i to point j. Total = exactly n lines.

Coordinate space (UI-SPEC §"Geometry"):
  - viewBox 0..1000 abstract units
  - center = (500, 500)
  - radius = 460
  - x_i = 500 + 460 * cos(θ_i); y_i = 500 + 460 * sin(θ_i)

Curated 8 pairs (UI-SPEC §"Curated (n, k) set — LOCKED"):
  [{n:200,k:2},{n:200,k:3},{n:240,k:5},{n:300,k:7},{n:360,k:11},{n:144,k:89},{n:180,k:179},{n:250,k:13}]

Vitest path-alias resolution (Revision 2026-06-04):
  - Vitest does NOT auto-resolve TS `paths` from tsconfig.json. The standard fix is the `vite-tsconfig-paths` plugin, configured via a minimal `vitest.config.ts`:

      import { defineConfig } from 'vitest/config'
      import tsconfigPaths from 'vite-tsconfig-paths'
      export default defineConfig({
        plugins: [tsconfigPaths()],
        test: { passWithNoTests: true },
      })

  - Test file uses explicit Vitest imports — no globals:

      import { describe, it, expect } from 'vitest'

  - This makes `npx tsc --noEmit` resolve the symbols via the installed `vitest` package types without any tsconfig.types or include changes; no globals; no tsc breakage.
