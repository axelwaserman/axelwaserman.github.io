## Spacing Scale

Tailwind v4 default 4-step scale (multiples of 4) is the baseline. Section-level vertical rhythm uses the existing fluid token `--space-section`. New tokens introduced in Phase 5: zero — Phase 5 must reuse the established scale. Lock the values below:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`) | Inline icon-to-text gap inside the "Download CV (PDF)" button; chord-coordinate decimal gap inside the (n,k) caption |
| sm | 8px (`gap-2`) | Pill spacing in skill-group items list; spacing between number-input label and field |
| md | 16px (`gap-4`, `mb-4`) | Default gap between hero CTA links (preserve Phase 2); space between WorkEntry bullets |
| lg | 24px (`gap-6`, `mb-6`) | Vertical breathing room between role description and bullet list; mandala-controls cluster gap |
| xl | 32px (`gap-8`, `mb-8`) | Skill-group cluster vertical gap; secondary download button top margin from last work entry |
| 2xl | 48px (`mb-12`) | Section-internal block separation (Work block → Education block → Skills block) — preserves Phase 2 D-07 internal rhythm even though outer layout re-centers per D-02 |
| 3xl | 64px (`py-16`) | Floor for section vertical padding on small viewports (covered by `--space-section` clamp) |
| `--space-section` | `clamp(4rem, 3rem + 5vw, 10rem)` | Top/bottom padding on each major section (Hero, CV, Projects, Contact). Already declared in `src/styles/tokens.css`. Re-centered sections in D-02 keep this. |

Exceptions:

- **Touch targets — 44×44 minimum (WCAG 2.5.5):** the mandala refresh icon-button and the (n,k) number inputs MUST hit at least 44×44 effective click area on touch devices (use `p-3` if rendered glyph is <20px). This is the only carve-out from the strict 4-step scale.
- **Mandala SVG viewBox padding:** SVG viewBox is 1000×1000 abstract units (see Mandala visual contract); on-screen pixel size is governed by container width, not the spacing scale.

## Typography

Two families locked: **Instrument Serif** for headings + the favicon monogram + the mandala (n,k) caption; **Sora** for body, UI, and number-input glyphs. No third family. Existing fluid type scale tokens in `tokens.css` are the source of truth — Phase 5 introduces no new sizes. Two weights locked across the phase: **regular (400)** + **semibold (600)**.

| Role | Token | Value | Family / Weight | Line Height |
|------|-------|-------|-----------------|-------------|
| UI / caption | `--text-ui` | `clamp(0.875rem, 0.82rem + 0.28vw, 1rem)` (~14–16px) | Sora 400 | 1.4 |
| Body | `--text-body` | `clamp(1rem, 0.92rem + 0.4vw, 1.125rem)` (~16–18px) | Sora 400 (description, bullets), Sora 600 (role, button label, skill-group category heading) | 1.5 (body), 1.6 (bullets) |
| Heading (section) | `--text-heading` | `clamp(1.75rem, 1.4rem + 1.75vw, 2.5rem)` (~28–40px) | Instrument Serif 400 | 1.2 |
| Hero display | `--text-hero` | `clamp(3rem, 1.5rem + 7.5vw, 7rem)` (~48–112px) | Instrument Serif 400 | 1.05 |

**Phase-5-specific rules:**

- **WorkEntry bullets** render at `--text-body` Sora 400 with line-height **1.6** (slightly looser than the role description's 1.5) so the multi-bullet block reads as a list, not a paragraph wall.
- **SkillGroup category heading** (e.g. "Engineering Leadership") renders at `--text-body` Sora **600**, in `--color-text` (NOT muted). Distinct enough to anchor the cluster; restrained enough to avoid competing with the Instrument-Serif section heading above.
- **SkillGroup items** render at `--text-ui` Sora 400 in `--color-text`.
- **Mandala (n,k) caption** ("n = 200, k = 5" rendered alongside the SVG) is `--text-ui` Sora 400 in `--color-muted`. Tabular numerals via `font-variant-numeric: tabular-nums` so swapping values does not re-flow the caption.
- **Number-input glyphs** are Sora 400 at `--text-ui`. Tabular numerals.
- **Refresh button label** is `aria-label` only (no visible text); icon glyph is centered.
- **"Download CV (PDF)" button label** is Sora **600** at `--text-ui`, sentence case, exact string `"Download CV (PDF)"` — always include the format hint.
- **Favicon "AW"** is Instrument Serif **400**, ~22pt within a 32×32 ImageResponse box (see Favicon contract). Letter-spacing **-0.02em** (slight negative tracking) so the two glyphs read as a monogram, not two letters.

## Color

Existing four-token parchment palette is locked from Phase 1. Phase 5 introduces **no new color tokens**. Anti-template policy (web/design-quality.md) is enforced by avoiding gradients on the mandala and avoiding a "brand-color" CTA on the download button.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (~70%) | `--color-surface` | `oklch(97% 0.01 75)` (warm parchment) | Page background, favicon background, hero/CV/Projects/Contact section backgrounds, mandala SVG background (transparent — surface bleeds through) |
| Secondary (~22%) | `--color-text` | `oklch(18% 0.01 75)` (near-black warm) | All body copy, headings, hero name, work role, education degree, skill items, favicon "AW" glyph, mandala stroke (default), download button text |
| Tertiary / muted (~5%) | `--color-muted` | `oklch(55% 0.03 75)` (warm gray) | Section labels (uppercase eyebrow), supporting metadata (company — dates, institution — years), bullet markers, mandala caption, number-input borders, refresh-icon stroke at rest, link-decoration default |
| Accent (~3%) | `--color-accent` | `oklch(62% 0.19 55)` (warm amber/terracotta) | **Strict reserved list — see below**. Never used decoratively. |

**`--color-accent` is reserved exclusively for these elements (and nothing else):**

1. Hover/focus link decoration on all anchor links (existing pattern in Hero / Contact)
2. Focus ring (`outline`) on every interactive element: hero CTAs, contact links, number inputs, refresh button, download button — `outline: 2px solid var(--color-accent); outline-offset: 2px`
3. The "Download CV (PDF)" button's **bottom underline border** at rest (1px hairline) — see Download Button visual contract
4. The refresh button's **icon stroke on hover/active** (default stroke is `--color-muted`)
5. The (n,k) number-input's **border on focus** (default border is `--color-muted` at 30% opacity)

**The mandala stroke is NOT accent.** It uses `--color-text` at 35% opacity (see Mandala visual contract). Using accent would push it toward stock decorative-blob territory and fail the anti-template policy.

**Destructive color:** none — no destructive actions exist in Phase 5. Number-input out-of-range validation uses **clamp-on-blur** (see Number Input contract), not error coloring.

**Contrast check (must pass in implementation):**
- `--color-text` on `--color-surface`: ~14.5:1 (well above WCAG AAA)
- `--color-muted` on `--color-surface`: ~4.6:1 (passes WCAG AA for body text)
- `--color-accent` on `--color-surface`: ~3.4:1 — **passes AA only as non-text/large-text**, used here exclusively as decoration (focus rings, underlines, icon strokes), never as body text. This is the deliberate anti-template choice.
- Mandala stroke (`--color-text` at 35% opacity, ≈ `oklch(18% 0.01 75 / 0.35)`) on parchment: effective ~3.0:1 — **passes WCAG AA for non-text graphical elements (1.4.11 = 3:1)**. Does not need to clear text contrast because the mandala is decorative; it is announced via `role="img"` + `aria-label`.

## Mandala Visual Contract (Times-Tables-on-a-Circle)

> This section is the centerpiece of Phase 5 and the highest-risk anti-template item. Every visual parameter is locked here.

### Algorithm (D-03)

For curated pair `(n, k)`:

1. Place `n` points evenly on a unit circle: point `i` at angle `2π·i/n − π/2` (start at 12 o'clock, clockwise sweep), for `i ∈ [0, n)`.
2. For each `i`, compute `j = (i × k) mod n`.
3. Render an SVG `<line>` from point `i` to point `j`. Total: exactly `n` lines.

Pure function: `generateLines(n: number, k: number): { x1: number; y1: number; x2: number; y2: number }[]`. Deterministic. No external math/animation library.

### Curated (n, k) set — LOCKED (D-04)

Exactly **8 pairs**. Each is included for a distinct visual quality. Planner picks one at random per page load (D-04) and on hero-scroll-out reset (D-06):

| # | n | k | Visual character | Why included |
|---|---|---|------------------|--------------|
| 1 | 200 | 2 | Cardioid | The canonical "Mathologer" first-impression pattern — single heart-shaped envelope. Sets the tone. |
| 2 | 200 | 3 | Nephroid (two-cusp) | Classic second-step in the family. Shows the algorithm produces multiple fundamentally different curves. |
| 3 | 240 | 5 | 4-petal star envelope | Dense but readable; petals form a cross axis. |
| 4 | 300 | 7 | 6-petal star | High point count, sharp star with fine inner web. |
| 5 | 360 | 11 | 10-petal dense star | Highest visual complexity in the set; near-tessellation. |
| 6 | 144 | 89 | Sparse pentagram-ish | Coprime with Fibonacci-adjacent numbers — produces a clean sparse polygonal weave. |
| 7 | 180 | 179 | Near-regular polygon (tight chords) | k = n−1 produces tiny chords near the circle edge — visually a textured ring, very different from the others. |
| 8 | 250 | 13 | Asymmetric rosette | Prime k with non-prime n — irregular but balanced; breaks the "always symmetric" pattern in the set. |

The set spans cardioid → nephroid → multi-petal stars → sparse weave → tight ring → asymmetric rosette. Strong visual range; no two pairs read as duplicates.

### Stroke (anti-template gate)

| Property | Value | Rationale |
|----------|-------|-----------|
| `stroke` | `var(--color-text)` (resolved to `oklch(18% 0.01 75)`) | Ink-on-parchment. Editorial. NOT accent (would feel like brand decoration). NOT a gradient (would feel like a stock SaaS blob). |
| `stroke-opacity` | `0.35` | Calm and atmospheric. Lets parchment breathe through. Passes WCAG 1.4.11 graphical contrast (~3.0:1). |
| `stroke-width` | `1` (in viewBox units, where viewBox is `0 0 1000 1000`) | Hairline. With 1000-unit viewBox rendered at ~500–700px, ends up as ~1.4–2px on screen. Editorial line weight. |
| `stroke-linecap` | `round` | Soft endpoints; reads as ink, not laser. |
| `fill` | `none` | Lines only. |
| Background | None — SVG is transparent; parchment surface bleeds through. | No box, no card, no shadow. |

### Geometry

| Property | Value |
|----------|-------|
| `viewBox` | `0 0 1000 1000` (square abstract grid) |
| Circle center | `(500, 500)` |
| Circle radius | `460` (40-unit margin on every side, prevents stroke clipping with `linecap="round"`) |
| `preserveAspectRatio` | `xMidYMid meet` (centers and never crops) |
| Outer container aspect | `1 / 1` (square; CSS `aspect-ratio: 1 / 1`) |
| Container max width on desktop | `min(50vw, 600px)` — fills hero right column (D-01) without ever exceeding 600px |
| Container width on mobile (<640px) | `min(70vw, 360px)` — see Mobile Layout below |

**No drop shadow, no `filter: blur()`, no glow.** Anti-template gate.

### Scroll-driven rotation (D-07)

| Property | Value |
|----------|-------|
| Property animated | `transform: rotate(...)` on the outermost `<svg>` element only |
| Compositor-only | YES (transform + opacity rules from `web/performance.md`) |
| Rotation rate | **1° per 8 pixels of scroll** (i.e. `rotation = (scrollY * 0.125) deg`) — calm, editorial; full hero scroll (~700px) yields ~87° rotation, almost a full quarter turn. Not gimmicky. |
| Frame loop | `requestAnimationFrame`. Scroll handler is `passive: true` and does **only** `latestScrollY = window.scrollY`; the rAF loop reads `latestScrollY` and writes `transform`. No layout reads inside the rAF. |
| `will-change` | `transform` set on the SVG **only while the hero is in the viewport**, removed via IntersectionObserver when the hero leaves. Avoids perma-promoted compositor layer. |
| Origin | `transform-origin: 50% 50%` (rotates around center) |
| Reduced-motion (D-08) | When `prefers-reduced-motion: reduce` is true (via the existing `useReducedMotion` hook): rotation disabled entirely; SVG renders static at 0° rotation. The rAF loop never starts. Inputs and refresh button still fully functional. |

### Hero scroll-out reset (D-06)

When the hero's bottom edge crosses above the viewport top (detected via IntersectionObserver, threshold = 0):

1. Pick a new random pair from the curated set (different from the currently displayed pair).
2. Update n + k state.
3. Re-render mandala.

When the user scrolls back up and the hero re-enters the viewport, they see the freshly chosen pair. Any custom (n,k) the user typed before scrolling away is discarded — this is intentional per D-06.

## Anti-Template Self-Audit

Verifying Phase 5 satisfies `web/design-quality.md`. Required: at least four qualities; this phase delivers seven:

1. **Clear hierarchy through scale contrast** — `--text-hero` (display Instrument Serif) over `--text-body` Sora 600 (title) over `--text-body` Sora 400 (bio) is a 3-step deliberate hierarchy; the same pattern repeats inside CV (Sora 600 role over Sora 400 description over Sora 400 bullets at looser leading).
2. **Intentional rhythm in spacing** — re-centered CV blocks use 48px between blocks, 32px between entries, 24px between eyebrow and first entry; not uniform padding everywhere.
3. **Depth or layering through motion** — scroll-driven mandala rotation (1° per 8px) introduces a physically anchored depth cue; SVG lines sit visually above the parchment without needing shadows.
4. **Typography with character and a real pairing strategy** — Instrument Serif (display, headings, favicon monogram, mandala-aria-label voice) paired with Sora (body, UI, controls). Locked since Phase 1.
5. **Color used semantically, not just decoratively** — accent is reserved for an explicit list (focus rings, link hover, mandala input focus, refresh icon hover, download button hairline). Never used as fill or for ambient decoration.
6. **Hover, focus, and active states that feel designed** — refresh button has translate + faint accent wash; download button has icon translateY + bottom-border expansion; number inputs have border-opacity step + accent focus border. Each is specific to the element's affordance.
7. **Grid-breaking editorial composition** — hero is two-column with mandala on the right; CV/Projects/Contact are centered (deliberate departure from Phase 2's two-column label-anchor layout). The site is no longer a uniform stack.

Banned-pattern audit:

- ❌ Stock hero with centered headline + gradient blob → **Avoided.** Hero is left-aligned (preserved); decorative element is an algorithmic mandala in stroke-only ink, not a gradient.
- ❌ Default card grids → **Avoided.** Project cards keep their Phase 3 anti-template treatment; the centered grid container does not regress this.
- ❌ Unmodified library defaults → **N/A.** No library is in use (no shadcn).
- ❌ Safe gray-on-white with one decorative accent → **Avoided.** Parchment is warm (oklch hue 75°), accent is reserved-list only, mandala stroke is ink at 35% opacity (not gray).
- ❌ Default font stacks → **Avoided.** Instrument Serif + Sora locked since Phase 1 with deliberate weight pairing.
